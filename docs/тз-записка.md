# 📕 Техническое задание — Travel World SPA

---

## 🎯 Цель проекта

Разработка SPA-приложения туристического агентства с собственной архитектурой без использования фреймворков (React / Vue / Angular).

Проект выполняет две задачи:

1. 📌 Демонстрация туристического сайта
2. 🧠 Исследование архитектуры SPA-фреймворка

---

## 🧱 Функциональные требования

### 📄 Страницы

Приложение должно содержать:

- Главная страница (/home)
- Страница "О нас" (/about)
- Страница предложений (/offers)
- Страница авторизации (/auth)

---

### 🔁 Навигация

- Переходы без перезагрузки страницы
- Использование History API
- Поддержка функции navigate()

---

### 🔐 Авторизация

- Ввод ключевого слова и купона
- Сохранение данных в localStorage
- Доступ к /offers только после авторизации

---

### 🎨 UI система

- Каждая страница имеет собственный CSS файл
- Стили подключаются динамически
- Исключаются конфликты между страницами

---

## 🧠 Архитектурные требования

### 📄 Page system

Каждая страница должна содержать:

- HTML разметку
- функцию mount (логика и события)
- функцию unmount (очистка ресурсов)

---

### 🧭 Router

Router должен:

- регистрировать страницы
- обрабатывать смену URL
- инициировать рендер страницы

---

### 🔁 Lifecycle

Каждая страница должна поддерживать:

- mount → инициализация
- unmount → очистка

---

## 📦 Ограничения

- без React / Vue / Angular
- только Vanilla JavaScript
- без backend
- данные только в localStorage

---

## 🚀 Результат

В результате должно получиться SPA-приложение с:

- навигацией без перезагрузки
- системой страниц
- авторизацией
- изолированными стилями
- базовой архитектурой фреймворка
