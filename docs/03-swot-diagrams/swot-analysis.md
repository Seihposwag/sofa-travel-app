## Стратегии на основе SWOT

### SO (Сила × Возможность)
Использовать чистую архитектуру проекта как основу для интеграции Django REST API: кастомный Router и сервисный слой `api.js` легко добавляются в существующую систему страниц.

### ST (Сила × Угроза)
Lifecycle (mount/unmount) и модульная структура уже минимизируют утечки памяти; при росте числа страниц архитектура остаётся предсказуемой благодаря единому контракту Page.

### WO (Слабость × Возможность)
Захардкоженные данные о турах заменяются вызовами к Django REST API без изменения структуры страниц — нужно лишь добавить слой `services/api.js`.

### WT (Слабость × Угроза)
Слабая клиентская авторизация + угроза очистки localStorage → приоритетно перейти на серверную JWT-аутентификацию при следующей итерации.
